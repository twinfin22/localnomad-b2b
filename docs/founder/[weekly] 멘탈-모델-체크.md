# Founder Mental Model Check (파운더 멘탈 모델 검증)

> **목적**: Gen의 머릿속 시스템 이해와 실제 시스템 사이의 gap을 찾아 교정합니다.
> **방법**: Gen이 자기 말로 "이 시스템이 어떻게 동작하는지" 적으면, Cowork이 틀린 부분을 교정합니다.
> **주기**: 주간 리뷰 시 (또는 Gen이 "이해가 안 된다"고 느낄 때 즉시)
> **마지막 검증**: (아직 미실시)

## 검증 방법

### Step 1: Gen이 자기 말로 적기

아래 질문에 대해 Gen이 직접 답을 적습니다. 코드 용어 없이, 비즈니스 언어로 적어도 OK.

### Step 2: Cowork이 교정

Gen의 답과 실제 시스템을 비교하여:
- ✅ 맞음 — 그대로 유지
- ⚠️ 부분적으로 맞음 — 어디가 다른지 교정
- ❌ 틀림 — 왜 틀렸는지 설명 + 실제 동작 방식 시각화

### Step 3: Gap 해소

틀린 부분에 대해 Cowork이 시각화(다이어그램)로 재설명. Gen이 "이제 이해했다"고 확인.

# Subsystem Owner Matrix (서브시스템 오너 매트릭스)

> **목적**: 시스템 각 영역의 책임자를 명확히 하여 "이 문제는 누가 해결하는가?"를 즉시 판단할 수 있게 합니다.
> **역할 정의**:
> - **Gen (Founder)**: 최종 의사결정권자. 스펙, 우선순위, 비즈니스 로직 판단.
> - **Cowork (PM/Architect Agent)**: 설계, 검증, 문서 유지보수, 워크플로우 관리.
> - **Claude Code (Coding Agent)**: 코드 작성, 빌드, 테스트 실행.
> **마지막 업데이트**: W2 D7

---

## 책임 흐름도

```
새 기능 요청이 들어오면:

  Gen ──→ "이 기능이 필요하다" (뼈대 + 성공 기준)
    │
    ▼
  Cowork ──→ "이렇게 만들면 된다" (상세 설계 + 영향 범위)
    │
    ▼
  Gen ──→ "이 설계가 맞다, 진행해" (승인)
    │
    ▼
  Claude Code ──→ 코드 구현 + 테스트
    │
    │  ⚠️ 의사결정 필요 시:
    │  Claude Code → Gen (옵션 + 장단점, 추천 없이)
    │  Gen → 결정 → decision-log.md 기록
    │
    ▼
  Cowork ──→ 검증 (빌드, 보안, 스펙 준수)
    │
    ▼
  Gen ──→ 최종 확인 (성공 기준 3가지 통과 여부)
```

## 비상 시 에스컬레이션

```
문제 발생 시 누가 처리하는가:

  빌드 실패 ──→ Claude Code가 자체 수정 ──→ 2회 실패 시 Cowork 개입
  보안 이슈 ──→ Cowork이 감지 ──→ 즉시 Gen에게 보고 ──→ Gen 판단
  스펙 불일치 ──→ Cowork이 감지 ──→ Gen에게 보고 ──→ Gen 판단
  외부 의존성 지연 ──→ Gen이 직접 관리 ──→ 일정 재조정
  Claude Code 무단 행동 ──→ Cowork이 감지 ──→ Gen에게 보고 ──→ 승인 또는 롤백
    (예: Playwright 무단 추가 사건)
```

# VisaCampus 랜딩페이지 기획안

> **랜딩페이지**: visacampus.org (별도 레포: `visacampus-landing/`, Vercel)
> **대시보드 앱**: app.visacampus.org (기존 레포: `localnomad-b2b/`, Vercel)
> **목적**: 대학 국제처 담당자 → 8주 무료 파일럿 신청 (리드 수집)

---

## 1. 목적과 전략

**핵심 전략**: "공감 → 해결 → 증거 → 행동" 4단계 설득

**타겟 페르소나**:
- 주 사용자: 국제처 주무관 (35-50세, 유학생 비자/FIMS 담당)
- 결정권자: 국제교류처 과장/팀장 (Risk Averse)
- 공통: 기술 친화성 낮음, 업무 스트레스 높음, IEQAS 규제 압박

**전환 목표**: 8주 무료 파일럿 신청 폼 제출 (이메일 + 소속 + 담당 업무)

**환경 요구사항**:
- 모바일 퍼스트 반응형 (모바일 ↔ 데스크톱)
- Windows 환경 최적화 (Segoe UI 폴백, ClearType 렌더링)
- Chrome / Edge 최적화 (주요 타겟 브라우저)

**파일럿 기간: 8주** — 대학 온보딩(2~3주) + 실사용(5~6주) + FIMS 보고 시즌 1회 포함 가능

## 2. 현재 구현 상태 (랜딩페이지에 반영)

| 기능 | 상태 | 랜딩페이지 표기 |
|------|------|----------------|
| 통합 학생 관리 대시보드 | ✅ 구현 | 별도 마킹 없이 표기 |
| Traffic Light 상태 관리 | ✅ 구현 | 별도 마킹 없이 표기 |
| 비자 만료 캘린더 + 알림 | ✅ 구현 | 별도 마킹 없이 표기 |
| 엑셀/CSV 대량 업로드 | ✅ 구현 | 별도 마킹 없이 표기 |
| IEQAS 불법체류율 게이지 | ✅ 구현 | 별도 마킹 없이 표기 |
| FIMS 호환 내보내기 | 🔜 개발 중 | "출시 예정" 배지 + 연한 글씨 |
| 변동신고 자동 감지 | 🔜 개발 중 | "출시 예정" 배지 + 연한 글씨 |
| AI 다국어 상담봇 | 🔜 개발 중 | "출시 예정" 배지 + 연한 글씨 |
| 카카오/SMS 알림 | 🔜 개발 중 | "출시 예정" 배지 + 연한 글씨 |
| PII 암호화 (여권/외국인등록번호) | ✅ 구현 | "(선택)" 표기 — 입력 자체가 optional |

**표기 규칙**: 사용 가능 기능은 따로 마킹 없이 정상 표기. 출시 예정만 "출시 예정" 배지 + 연한 글씨색(text-gray-400)으로 구분. 시계 이모지 사용하지 않음. 섹션 내에서 출시 예정 항목은 뒤에 배치.

## 3. 페이지 구조 (6개 섹션)

### 섹션 1: Hero (첫 화면)

**역할**: 3초 안에 "이건 나를 위한 서비스다" 인지

```
[헤드라인] — 아래 후보 중 택 1 (별도 섹션에서 논의)

[서브헤드]
FIMS 보고부터 비자 만료 관리까지, 한 곳에서.
대학 국제처를 위한 유학생 관리 플랫폼

[기능 요약 3개 — 가로 배치, 수치 없이]
📊 유학생 현황 통합 조회  |  🔍 IEQAS 이탈률 실시간 추적  |  📋 FIMS 보고 간소화

[CTA 버튼]
"8주 무료 파일럿 신청" (Primary)
[보조 텍스트] 설치 없이 바로 시작 · 8주간 무료
```

**디자인 노트**:
- "SaaS" 대신 "유학생 관리 플랫폼" 사용
- 수치 주장 없음 — 기능 설명만
- 히어로 헤드라인 확정: **"유학생 비자 관리, 엑셀에서 벗어나세요"**

### 섹션 2: Problem (공감)

**역할**: "이 사람들이 내 상황을 정확히 안다"

```
[섹션 제목]
"이런 고민, 매 학기 반복되고 계시죠?"

[4개 Pain Point 카드 — 부제 없이 제목만]

카드 1: 📋 "FIMS 정기보고마다 며칠이 사라집니다"
  - 매 분기 엑셀 → FIMS 수동 입력
  - 입력 오류가 나면 처음부터 다시 확인

카드 2: ⚠️ "변동신고 15일 기한, 놓치면 과태료입니다"
  - 자퇴·제적·미등록 변동 실시간 파악 불가
  - 수기 관리로 누락 발생

카드 3: 🌏 "유학생 상담, 같은 질문을 하루에도 수십 번 반복합니다"
  - 언어 장벽, 비자 절차 반복 질문
  - 학기초에 업무 폭증

카드 4: 😰 "2025~26년, 44개 대학이 비자 발급 제한을 받았습니다"
  - IEQAS 불법체류율 2% 초과 시 인증 위험
  - 실시간 모니터링 수단 부재
```

### 섹션 3: Solution (해결)

**역할**: "이 도구가 정확히 그 문제를 해결한다"

```
[섹션 제목]
"VisaCampus가 도와드립니다"

[3개 핵심 기능]

기능 1: 통합 학생 관리 대시보드 (마킹 없음)
  - 한 화면에 전체 유학생 현황
  - Traffic Light: 초록(정상) → 노랑(주의) → 빨강(위험)
  - IEQAS 불법체류율 실시간 게이지
  - 비자 만료 캘린더 + 알림
  [스크린샷: 대시보드 화면]

기능 2: FIMS 보고 간소화 — "출시 예정" 배지
  - 엑셀 대량 업로드 + AI 컬럼 자동 매핑 (사용 가능, 위에 배치)
  - 정기보고(4회/년): FIMS 호환 엑셀 자동 생성 (연한 글씨 + "출시 예정")
  - 변동신고: 학생 상태 변경 시 자동 감지 + 15일 카운트다운 (연한 글씨 + "출시 예정")
  [스크린샷: Import 화면]

기능 3: AI 다국어 상담봇 — 출시 예정 🔜
  - 한국어·영어·중국어·베트남어·우즈베크어·몽골어
  - 비자 연장 절차, 필요 서류, 기한 안내 자동 응답
  - 복잡한 질문은 담당자에게 자동 전달
  [목업 이미지]
```

### 섹션 4: Before → After 비교

**역할**: "도입 전후 차이를 극적으로 시각화"

```
[섹션 제목]
"이렇게 달라집니다" (부제 없음)

[Before/After 가로 분할 카드 — 사용 가능 항목 먼저, 출시 예정 뒤에]

— 사용 가능 (진한 색상) —

카드 1: 학생 현황 파악
  Before (빨간 배경): 엑셀, FIMS, 학사시스템 — 3곳을 따로따로 열어서 확인
  After (초록 배경): 한 화면에서 전체 학생 상태 + 위험도 즉시 확인

카드 2: IEQAS 이탈률 관리
  Before: 월말에 수작업으로 계산, 누락되면 인증 위험
  After: 대시보드에서 실시간 게이지로 즉시 확인

— 출시 예정 (연한 색상, opacity 80%) —

카드 3: FIMS 정기보고
  Before: 엑셀 → FIMS 수동 입력, 분기마다 며칠 소요
  After: 대시보드에서 FIMS 호환 파일 바로 생성 + "출시 예정" 배지

카드 4: 유학생 상담
  Before: 반복 질문에 수동 응대, 6개 언어 장벽
  After: AI 챗봇이 다국어 기본 상담 자동 처리 + "출시 예정" 배지
```

**디자인 노트**:
- 카드 좌우 분할 (Before=빨간 배경 / After=초록 배경) — 극적 대비
- 모바일에서는 상하 분할
- 사용 가능 항목: 진한 색상 / 출시 예정 항목: 연한 색상 + opacity
- 수치 없음, 부제 없음

### 섹션 5: 안심 요소

**역할**: "데이터가 안전하다"

```
[섹션 제목]
"개인정보, 안심하세요"

[2개 항목만]

1. 🔒 개인정보 보호
  - AWS 서울 리전 (국내 데이터 보관)
  - 여권번호·외국인등록번호 AES-256 암호화 (입력은 선택사항)
  - 개인정보 접근 이력 자동 기록

2. 👤 역할별 접근 권한
  - 관리자 / 매니저 / 열람자 3단계 권한 분리
  - 민감 정보는 권한에 따라 마스킹 처리

[데이터 엑스포트 보장 — 안심 섹션 하단]
"무료 체험이 끝나도 데이터는 언제든 내보낼 수 있습니다."
"엑셀/CSV 형식으로 전체 데이터 엑스포트를 지원합니다."
```

**변경 사항 (Gen 피드백 반영)**:
- "법적 안정성" 섹션 삭제 — 변호사 검증 전이므로 자신할 수 없음
- "행정사법 저촉 없음" 삭제
- PII 암호화에 "(입력은 선택사항)" 명시
- 파일럿 대학 Trust signal 삭제 — 아직 진행 전

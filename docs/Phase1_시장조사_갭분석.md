|---------|------|--------|

#### 7. 기술 검증 (Technical Feasibility)

##### 7.1 AI 컬럼 매핑 정확도

**현황**
- Phase 1 스펙: "AI 컬럼 매핑 엔진" 정확도 80% 목표
  > "F1-04 엑셀/CSV 임포트 + AI 컬럼 매핑 | P0 (필수) | W2"
  > "파일럿 대학 엑셀로 매핑 정확도 80%+ 달성"

- **80% 정확도의 기준이 무엇인가?**
  - 엑셀 헤더 자동 감지인가? (예: "학번", "Student ID" → student_id 필드)
  - 데이터 타입 자동 감지인가? (예: 숫자 컬럼 → integer)
  - 필드 의미 자동 감지인가? (예: "2026-02-13" → visa_expiry)

**미검증 항목**
1. 한글/영문 혼합 엑셀 → Claude API 처리 능력
   - 대학 엑셀은 "학번", "이름", "성명(영문)", "비자만료", "保険失効" 섞여 있음
   - Claude가 문맥에서 필드 의미를 올바르게 파악할 수 있는가?

2. 데이터 오류 처리
   - "비자만료: 2026/13/45" (오타) → 감지 능력?
   - "학번: ABC123" (잘못된 형식) → 경고?
   - False positive 발생 시 사용자 부담?

3. 매핑 정확도의 정의
   - "담당자가 수정 없이 바로 임포트 가능" → 100% 기준
   - "담당자가 3개 이상의 필드만 수정하면 됨" → 80%?
   - 누가 이를 검증하는가?

**권장 조치**
1. **W1 AI 엔지니어 태스크**
   - [ ] 파일럿 대학으로부터 최소 3종류의 실제 엑셀 수집 (W0 완료)
   - [ ] Claude API를 사용한 컬럼 매핑 프로토타입 구축 (24시간)
   - [ ] 3가지 엑셀에 대해 정확도 측정:
     - 정답: 담당자가 수동으로 매핑
     - 예측: Claude가 매핑
     - 정확도 = (일치 필드 수) / (전체 필드 수)
   - [ ] 80% 미만이면 → 프롬프트 튜닝 or 규칙 기반 보강

2. **매핑 정확도 벤치마크 수립 (W2 D4)**
   - [ ] "정확도 80%" 정의를 명확히
   - [ ] 핸들링할 수 없는 케이스 명시 (예: "구 시스템 엑셀" 등)
   - [ ] 담당자 검증 단계 필수 (임포트 전 확인 화면)

**영향**: 매핑 정확도 미달 → 담당자 수작업 증가 → 사용성 저하 → NPS 하락

##### 7.2 저자원 언어 챗봇 품질

   - [ ] 1% 미만이면 → MVP에서 제외, Phase 2로 연기

**영향**: 저자원 언어 챗봇 실패 → 해당 국가 학생 불만 → NPS 저하 (다만 파일럿 대학이 해당 학생 많지 않으면 영향 작음)

#### 9. 금융 모델 (Financial Model)

**현황**
- Phase 1 스펙: 5개 파일럿 대학, 2000명+ 유학생 등록
- **CAC (Customer Acquisition Cost), LTV (Lifetime Value), 번인율 계산 전무**

**필수 정보 갭**

| 지표 | 정의 | 현황 | 필요 이유 |
|------|------|------|---------|
| CAC | 고객 1개 획득 비용 | ❌ | 마케팅 예산 결정 |
| LTV | 고객의 평생 수익 | ❌ | 단위 경제 검증 |
| Payback Period | CAC 회수 기간 | ❌ | 확장 가능성 판단 |
| Churn Rate | 월/년 고객 이탈률 | ❌ (가정: 5%?) | 성장 예측 |
| MRR/ARR | 월/연 경상 수익 | ❌ | 재무 계획 |
| Burn Rate | 월 지출 | ⚠️ (부분) | 런웨이(자금 소진 시점) |

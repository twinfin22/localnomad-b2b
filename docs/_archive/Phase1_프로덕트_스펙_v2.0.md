## 1. Phase 1 목표 및 성공 기준

### 1.1 목표

하이어비자(HireDiversity, 105개 대학 협약, 60억 투자)가 제공하는 핵심 기능을 모던 UX로 재구현하고, AI 자동화로 하이어비자가 못하는 것(IEQAS 불법체류율 실시간 추적, 다국어 AI 상담, FIMS 자동화)을 추가하여 **아산권 3개 대학(호서대·선문대·순천향대) 파일럿을 확보**한다. 아산권은 IEQAS 우수인증 대학이 집중되어 있으며(선문대 13년 연속, 호서대 12년 연속), 지리적 클러스터 효과로 영업 효율이 극대화되는 전략 거점이다.

**시장 컨텍스트:**
- 전체 유학생 272,573명(2025), 불법체류 약 36,000명(전체 대비 17.64%)
- 비수도권 유학생 비중 45.8%(상승세) — 지방대 타겟 전략의 근거
- 주요 출신국: 중국 30.2%, 베트남 29.7%, 우즈베키스탄 6.2%, 몽골 6.0% → AI 챗봇 언어 우선순위 근거
- 정부 목표: Study Korea 300K (2027년까지 유학생 30만명 유치)
- IEQAS: 4년 주기 평가, 학위과정 158개 + 어학연수과정 103개 인증
- 규제 강화: 2025년 24개 + 2026년 2학기 추가 20개 대학 비자 제한 예정

```
function calculateStatus(student):
  days_to_visa_expiry = student.visa_expiry - today()

  // ⚫ 연락두절: 최근 90일간 알림 응답 없음 + 출석률 0%
  if no_response_90days(student) AND student.attendance_rate == 0:
    return BLACK

  // 🔴 긴급 (하나라도 해당 시)
  if days_to_visa_expiry <= 30:
    return RED

    return RED
  if student.visa_status == EXPIRED:
    return RED
  if student.attendance_rate < 50:
    return RED
  if pending_fims_report(student) AND fims_deadline_days <= 3:
    return RED

  // 🟡 주의 (하나라도 해당 시)
  if days_to_visa_expiry <= 60:
    return YELLOW
  if student.enrollment_status == 휴학:
    return YELLOW
  if student.insurance_status IN [EXPIRING, EXPIRED]:
    return YELLOW
  if student.attendance_rate < 70:
    return YELLOW
  if address_change_unreported(student):
    return YELLOW
  if student.part_time_permit_expiry AND
     student.part_time_permit_expiry - today() <= 30:
    return YELLOW
  if pending_fims_report(student):
    return YELLOW

  // 🟢 정상
  return GREEN
```

| 단체 접수 미비 서류 | 학생 | 접수 마감 14/7일 전 | 카카오 + SMS + 이메일 |

**학생 알림 다국어 지원:**

- 한국어, 영어, 중국어, 베트남어, 우즈베크어, 몽골어
- 학생 프로필의 nationality 기반 자동 언어 선택
- 담당자가 학생별 선호 언어 수동 지정 가능

**알림 템플릿 예시 (베트남어):**

```
[로컬노마드] Xin chào {student_name},
Visa D-2 của bạn sẽ hết hạn vào ngày {visa_expiry}.
Vui lòng chuẩn bị các giấy tờ cần thiết để gia hạn visa.
Chi tiết: {dashboard_link}
```

**Acceptance Criteria:**

- [ ] 모든 알림 유형이 설정된 시점에 자동 발송
- [ ] 담당자용 알림은 대시보드 인앱 + 이메일 + 카카오 알림톡
- [ ] 학생용 알림은 카카오 알림톡 + SMS + 이메일 (모국어)
- [ ] 알림 발송 이력 전체 조회 가능
- [ ] 알림 읽음/미읽음 상태 추적 (담당자 대시보드)
- [ ] 카카오 알림톡 템플릿 사전 등록 및 검수 통과 필요 (W1에 신청)

## 4. Feature 2: AI 다국어 유학생 상담 봇

### 4.1 아키텍처

```

                                              │
                   ┌──────────────┐    ┌──────▼───────┐
                   │  답변 생성    │←───│  벡터 DB     │
                   │  (Claude API) │    │  (Pinecone)  │
                   └──────┬───────┘    └──────────────┘
                          │
                   ┌──────▼───────┐
```

### 4.2 지원 범위

**다국어 지원 등급:**

- 한국어 (A+ 95점), 영어 (A+ 96점), 중국어 (A 92점), 베트남어 (A 91점) → **즉시 배포 가능**
- 우즈베크어 (C+ 72점), 몽골어 (C 68점) → **조건부 배포** (전문가 검수 후, "AI 번역 참고용" 면책 표시 필수)
- 우즈벡/몽골어 주요 문제: 문법 오류, 전문 용어 부족, 직역식 표현, 러시아어 영향
- 학생 프로필의 nationality 기반 자동 언어 선택
- 담당자가 학생별 선호 언어 수동 지정 가능
- low-resource 언어(우즈벡/몽골) 응답에는 자동으로 "⚠️ AI 번역 참고용입니다. 정확한 정보는 국제교류처에 문의하세요" 면책 문구 추가
- **우즈베크어 주요 문제**: 러시아어 영향 문법 혼용, 비자 전문 용어 부재, 직역식 표현
- **몽골어 주요 문제**: 키릴 문자 혼용, 법률 전문용어 부족, 구어체/문어체 혼재
- **개선 로드맵**: Phase 2에서 비자 전문 용어 사전 구축 + 우즈벡/몽골 네이티브 검수자 확보 계획

**In Scope (답변 가능):**

| 카테고리 | 예시 질문 |
|----------|----------|
| 비자 연장 절차 | "D-2 비자 연장하려면 뭐가 필요해요?" |
| 구비 서류 안내 | "체류자격변경 서류 목록 알려주세요" |
| 체류지 변경 신고 | "이사하면 어디에 신고해야 하나요?" |
| 시간제취업 | "아르바이트 하려면 허가 받아야 하나요?" |
| 건강보험 | "외국인 건강보험 가입 방법이요" |
| 출입국관리사무소 위치/운영 | "서울출입국 주소가 어디예요?" |
| 대학 내규 (FAQ) | "우리 학교 유학생 장학금 조건이 뭐예요?" |

**Out of Scope (에스컬레이션):**

| 유형 | 대응 |
|------|------|
| 개별 사안 법적 판단 | "제가 비자 연장이 거절될까요?" → 에스컬레이션 |
| 허가 가능성 판단 | "E-7으로 변경 가능할까요?" → 에스컬레이션 |
| 불법체류 관련 | "비자 만료됐는데 어떡하죠?" → 긴급 에스컬레이션 + 담당자 알림 |
| 개인정보 요청 | "제 외국인등록번호 알려주세요" → 거부 |

### 4.4 에스컬레이션 로직

```
1. AI가 답변 불가 판단 시:
   - 학생에게: "이 질문은 담당자 확인이 필요합니다.
              국제교류처에 문의가 전달되었습니다."

2. 티켓 자동 생성:
   - 학생 정보 + 질문 내용 + AI 분석 요약
   - 우선순위 자동 판정 (긴급/일반)

3. 담당자 대시보드에 티켓 표시:
   - 미처리 티켓 카운트 뱃지
   - 클릭 시 학생 프로필 + 대화 이력 확인 가능

4. 담당자 응답:
   - 대시보드에서 직접 답변 입력
   - 답변이 학생 채팅으로 전달
```

### 4.5 접속 채널

| 채널 | 구현 방식 | 우선순위 | 비고 |
|------|----------|---------|------|
| 웹 채팅 위젯 | 대시보드 내 임베드 또는 별도 URL | P0 (W3) | 6개국어 지원 |
| 카카오톡 채널 | 카카오 비즈메시지 API (OAuth 2.0) 연동 | P1 (W4) | 템플릿 승인 2-3일 소요, 한국어/영어만 직접 지원 |
| SMS 폴백 | 카카오 미수신 시 SMS 자동 전환 | P1 (W4) | 외국 번호 학생용 필수 (카카오는 한국 번호만 지원) |

**카카오 알림톡 기술 상세:**
- 메시지 최대 1,000자 제한
- API 연동: BizMessage Agent 설치 방식 또는 REST API 방식 선택 (OAuth 2.0 인증)
- 중계 플랫폼: NHN Cloud 권장 (API v3.0, 건당 과금)
- 국제 SMS 대안: Twilio 또는 Infobip (KakaoTalk + SMS 통합)
- 템플릿 거절 사유: 광고성 메시지, 불명확한 비즈니스 목적, 정책 위반 내용
- 언어별 템플릿 별도 검수 필요 → 한국어/영어 우선 등록, 기타 언어 SMS 폴백
- 비용: SMS 대비 저가 (정확한 단가는 계약 규모에 따라 상이)

**Acceptance Criteria:**

- [ ] 6개국어 입력 인식 및 해당 언어로 응답
- [ ] FAQ 기반 질문 80%+ 자동 응답
- [ ] 에스컬레이션 시 담당자 대시보드에 티켓 생성
- [ ] 응답 시간 5초 이내
- [ ] 법적 판단, 허가 가능성 등 Out of Scope 질문 자동 필터링
- [ ] 대화 이력 저장 및 담당자 열람 가능
- [ ] 고유식별정보 요청 시 자동 거부

## 5. Feature 3: 데이터 보존 및 인수인계 시스템

### 5.1 범용 데이터 마이그레이션 도구

**지원 소스 유형:**

| 소스 | 설명 | 매핑 방식 |
|------|------|----------|
| 하이어비자 내보내기 | 하이어비자에서 추출한 엑셀/CSV | AI 자동 매핑 (사전 학습된 하이어비자 필드 구조) |
| 수기/엑셀 관리 | 대학마다 다른 자체 엑셀 양식 | AI 자동 매핑 + 담당자 확인 |
| FIMS 역임포트 | FIMS 입력용으로 정리해둔 엑셀 | AI 자동 매핑 (FIMS 양식 기반) |
| 복수 파일 병합 | 학기별/학과별 분산 파일 | AI 중복 감지 + 최신 데이터 우선 |

**마이그레이션 프로세스:**

```
1. 소스 유형 선택 (또는 자동 감지)
2. 파일 업로드 (복수 파일 가능)
3. AI 자동 매핑 + 담당자 확인
4. 데이터 검증 (누락/형식오류/중복)
5. 검증 리포트 확인 및 오류 수정
6. 임포트 실행
7. 마이그레이션 완료 리포트 생성
```

### 5.2 인수인계 시스템

**담당자 교체 워크플로우:**

```
1. 기존 담당자가 "인수인계 시작" 클릭
2. 신규 담당자 계정 생성 (학교 이메일 인증)
3. 권한 이관: 기존 담당자 → 신규 담당자
4. 인수인계 브리핑 자동 생성:
   - 현재 긴급 조치 필요 학생 목록
   - 진행 중인 비자 연장/변동신고 건
   - 최근 30일 변동 사항 요약
   - FIMS 정기보고 다음 마감일
   - 단체 접수 진행 현황
5. 기존 담당자 계정은 30일간 읽기 전용 유지 후 비활성화
```

**데이터 이식성:**

- 전체 데이터 CSV/JSON 내보내기 상시 가능
- 해지 시 데이터 반출 무료 지원
- 해지 후 6개월간 데이터 열람 가능
- 6개월 경과 후 복구 불가능한 방식으로 파기

**Acceptance Criteria:**

- [ ] 하이어비자/수기엑셀/FIMS 역임포트/복수파일 병합 모두 지원
- [ ] 마이그레이션 검증 리포트 자동 생성
- [ ] 담당자 교체 시 원클릭 권한 이관
- [ ] 인수인계 브리핑 자동 생성
- [ ] 전체 데이터 CSV/JSON 내보내기
- [ ] 감사 로그 영구 보관

### 9.2 법적 포지셔닝 가이드라인 (2차 조사 반영)

**삼쩜삼 판례 기반 SaaS 합법성 근거:**
- 삼쩜삼(세무사법 위반 혐의) → 경찰 불송치(2022.8) → 지검 불기소(2023.11) → 대검 재항고 기각(2025.5)
- 핵심 법리: "도구 제공" ≠ "전문 직업 행위". 사용자/전문가가 실제 업무 수행, SaaS는 도구에 불과
- **로컬노마드 적용**: 대학 담당자가 FIMS 입력·제출 수행, 로컬노마드는 데이터 정리·양식 생성 도구

**행정사법 경계선 준수:**
- ✅ 허용: 서류 양식 생성, 체크리스트 제공, 워크플로우 자동화, 일반 정보 안내
- ❌ 금지: 행정사 직인/서명 문서 작성, 대리 제출, 개별 사안 법적 판단
- AI 챗봇: 일반 절차 안내 가능, 특정 상황 법적 의견("비자 연장 거절될까요?") 불가 → 에스컬레이션

**개인정보보호법 준수:**
- 대학 = 개인정보처리자(Controller), 로컬노마드 = 수탁자(Processor)
- 위탁계약 필수 (개인정보보호법 제26조): 처리 목적, 범위, 기간, 파기 절차 명시
- 고유식별정보(여권번호, 외국인등록번호): AES-256 암호화 필수
- 과징금 위험: 위반 시 최대 5억원

**ISMS-P 인증 계획:**
- 비용: 약 900만원 (간편인증 기준, 600~1,100만원 범위)
- 소요 기간: 6개월+
- 심사 항목: 101개
- 전략: Phase 1에서는 ISMS-P 수준의 보안 설계를 적용하되, 공식 인증은 Phase 2 초반에 착수 (M6~M9)
- 유효기간: 3년 (갱신 심사)

**출입국관리법 신원보증 규정:**
- 제90조: 대학 총장 = 유학생 신원보증인 (신원보증기간 4년 한도)
- 제89조: 신원보증 철회 시 체류허가 취소/변경 가능
- 시행규칙 제77조: 신원보증 관련 세부 절차 규정
- **로컬노마드 적용**: 대시보드의 불법체류율 관리가 곧 대학 총장의 신원보증 의무 이행을 지원하는 도구

**삼쩜삼 추가 리스크 교훈:**
- 2024.12 개인정보보호위원회: 홈택스 간편인증 과정 개인정보 무단 수집 → 시정명령
- 2024.11, 2025.5: 세무대리 소개·알선 혐의로 추가 고발 진행 중
- **로컬노마드 교훈**: ① 개인정보 수집 시 명시적 동의 절차 철저히 설계 ② "도구 제공" 포지셔닝 문서화 ③ 행정사 업무 대리 절대 금지

**클라우드 위탁자 판단 기준:**
- AWS가 개인정보에 실제 접근 불가 → 수탁자 아님 (위탁계약 불필요)
- 로컬노마드 서버에서 개인정보 처리 → 로컬노마드 = 수탁자 (대학과 위탁계약 필수)
- 판단 기준: "배타적 권리자"가 누구인가에 따라 결정

**국외 데이터 이전 (개인정보보호법 제18조):**
- AWS 서울 리전 사용 → 국외 이전 이슈 회피
- Claude API(해외 서버) 호출 시 → 고유식별정보(여권번호, ARC) 마스킹 필수 (Section 9.1 AI 마스킹과 연동)
- 국외 이전 시 정보주체 동의 또는 동등한 보호 수준 국가 요건 충족 필요

**위탁계약 필수 조항 (개인정보보호법 제26조 + 시행령 제28조):**
1. 위탁 업무 범위 한정 (비자 관리, 알림 발송, 보고서 생성)
2. 위탁 업무 외 개인정보 처리 금지
3. 기술적·관리적 보호조치 (AES-256, TLS 1.3, 접근제어)
4. 재위탁 제한 (사전 서면 동의 필수)
5. 연 1회 이상 처리현황 점검 의무
6. 계약 종료 후 개인정보 파기 절차 (복구 불가능한 방식)
7. 위반 시 손해배상 조항 명시

### 9.3 개인정보 처리 흐름

```
[대학 담당자]
    │ 엑셀 업로드 (HTTPS/TLS 1.3)
    ▼
[API Server]
    │ 고유식별정보 감지 → AES-256 암호화
    ▼
[PostgreSQL (RDS)]

    │
    ├──→ [AI 챗봇 호출 시]
    │        │ 고유식별정보 마스킹
    │        ▼
    │    [Claude API] ← 마스킹된 데이터만 전달
    │
    ├──→ [FIMS 내보내기 시]
    │        │ 복호화 → 엑셀 생성 → 담당자만 다운로드

    │
    └──→ [데이터 파기]

```

```
W2: AI 매핑 엔진 ──→ W4: 마이그레이션 도구         │
    임포트 기능 ───→ W3: 변동신고 (diff 비교 필요)  │
                                                   │
W3: FIMS 내보내기 ←─────────────────────────────────┘
    (W0 양식이 없으면 개발 불가)
```
